# 因子工厂 vs 业内量化实践差异清单

> 2026-06-07 整理。覆盖 v1 因子工厂的量化逻辑决策点，与 Barra / 券商研报 / 学术惯例的对比。

---

## 一、中性化

| 细节 | 我们的做法 | 业内常见 | 差异影响 |
|------|-----------|---------|---------|
| **顺序** | z-score → 回归取残差 → re-standardize | Barra: 回归取残差 → z-score | 小，相关性通常 0.95+ |
| **回归权重** | OLS（等权） | Barra 用 WLS（sqrt(mv) 加权） | 中，大市值股票在中性化中被同等对待 |
| **行业分类** | 申万一级 | Barra 用自有/GICS；国内也用申万 | 小（国内市场适配合理） |
| **行业编码** | `pd.get_dummies(drop_first=True)` | 标准做法 | 无差异 |
| **回归求解** | `np.linalg.lstsq` | 标准做法；statsmodels 仅用于推断 | 无差异 |

OLS 等权回归意味着一个 50 亿市值的小票和一个 5000 亿的大票在中性化中权重相同。Barra 用市值加权是为了让模型更贴合组合风险（持仓也是市值加权的），但对因子研究阶段的影响不大，更多影响组合优化阶段。

## 二、缩尾（Winsorization）

| 细节 | 我们的做法 | 业内常见 | 差异影响 |
|------|-----------|---------|---------|
| **方法** | 仅 MAD（n=3, 常数 1.4826） | MAD 和 percentile 都常用 | 小 |
| **percentile 极限** | 配置文件里写了 `limits: [0.01, 0.99]` 但**代码没用** | Barra 通常用 0.5%/99.5% | 中，代码实际没有实现 percentile 方案 |
| **常数** | 1.4826 | 标准值 | 无差异 |
| **截面** | 逐日截面内做 | 标准做法 | 无差异 |

MAD 对非正态分布更稳健，percentile 更直观可控。配置里有 limits 但 `preprocessing.py` 只调了 `winsorize_mad`，等于 percentile 方案有入口没落地。

## 三、标准化（Z-score）

| 细节 | 我们的做法 | 业内常见 | 差异影响 |
|------|-----------|---------|---------|
| **ddof** | 全程 `ddof=0`（总体标准差） | ddof=0 和 ddof=1 都有 | 极小，大截面下几乎无差别 |
| **截面** | 逐日截面内做 | 标准做法 | 无差异 |
| **常数截面处理** | std=0 时返回全 0 | 各家的 fallback 不同 | 小 |

## 四、股票池筛选（Universe）

| 细节 | 我们的做法 | 业内常见 | 差异影响 |
|------|-----------|---------|---------|
| **ST 识别** | SQL 名称匹配 `name LIKE 'ST%'` | 用专门的 ST 状态字段 | 中，名称匹配有遗漏风险 |
| **新股剔除** | `new_stock_window_days: 20`（配置）/ `min_listing_days: 60`（配置）/ `250`（runner 默认） | 通常 60-120 个交易日（3-6 个月） | 中，A 股新股效应强，20 天太短 |
| **涨跌停处理** | 仅过滤开盘涨停（买）/开盘跌停（卖），无盘中涨停判断 | Barra 处理更细（区分涨跌停幅度等） | 小 |
| **板块限制** | 仅主板 `require_main_board: true` | 看研究目的，全市场更常见 | 取决于研究阶段 |
| **成交额门槛** | `min_amount: 1000`（千元 = 100 万元） | 差异大，通常看成交额分位数 | 小 |

## 五、未来收益计算

| 细节 | 我们的做法 | 业内常见 | 差异影响 |
|------|-----------|---------|---------|
| **公式** | `fwd_h = adj_close[t+h+1] / adj_close[t+1] - 1` | 标准做法 | 无差异 |
| **entry 价** | t+1 close（隔日收盘） | 多数用 t+1 close 或 t+1 open/VWAP | 小 |
| **收益类型** | 简单收益率 | 简单和 log 都有 | 极小（短期差距微弱） |
| **极端值处理** | forward return **没有缩尾** | 很多 shop 对 forward return 也缩尾 | 中，极端收益会扭曲 IC 和分层收益 |

Forward return 不缩尾是一个值得注意的点——某天某票涨 20%（涨停），它会在分层收益和 IC 回归里产生一个 outlier。很多量化团队的 pipeline 里 forward return 这端也会做一遍 winsorize。

## 六、IC 计算

| 细节 | 我们的做法 | 业内常见 | 差异影响 |
|------|-----------|---------|---------|
| **IC 类型** | Rank IC (Spearman) + Pearson IC 都算 | 标准做法 | 无差异 |
| **IC IR** | `mean_ic / std_ic`（ddof=0） | 标准做法 | 无差异 |
| **IC t-stat** | `mean / (std / sqrt(n))` | 标准做法 | 无差异 |
| **IC 截面最少股票** | `min_stocks >= 10` | 通常 30-50 | 小 |
| **Rolling IC 窗口** | 252 天 | 标准做法 | 无差异 |

## 七、分层与分组

| 细节 | 我们的做法 | 业内常见 | 差异影响 |
|------|-----------|---------|---------|
| **分组数** | 5 组 | 5 或 10 组 | 小 |
| **分组方式** | `pd.qcut` on rank（等数量分组） | 标准做法 | 无差异 |
| **组内加权** | **等权**（`mean(forward_return)`） | 很多 shop 同时报告等权和市值加权 | 中 |
| **单调性判断** | `is_monotonic_increasing` / `decreasing` | 更常用 Q5-Q1 spread 的统计显著性 | 小 |

等权分组收益容易受小票主导。市值加权版本能检验因子在大票上是否也有区分度——这是买方比较看重的。

## 八、回测

| 细节 | 我们的做法 | 业内常见 | 差异影响 |
|------|-----------|---------|---------|
| **回测范围** | Q5 等权多头 only | 标准做法是同时看多头、多空、以及纯因子组合 | 中 |
| **调仓频率** | 5 天 | 5 天合理，但月度因子应该匹配其信号周期 | 小 |
| **交易成本** | 0.1% 单边，线性扣减 | 0.1% 合理，但加上印花税（0.05%）+ 冲击成本才是完整的 | 小 |
| **换手公式** | `1 - |prev ∩ curr| / |curr|` | 标准做法 | 无差异 |
| **净值曲线** | 仅对调仓日逐点复利，中间用 ffill | 标准做法 | 无差异 |
| **多空年化** | `mean(spread) * 252` / `std(spread) * sqrt(252)` | 标准做法 | 无差异 |

目前回测只有 Q5 多头。多空价差（Q5-Q1）虽然算了累计收益和回撤，但没有形成独立的模拟组合曲线。纯因子组合（factor mimicking portfolio）完全没有做。

## 九、OOS 检验

| 细节 | 我们的做法 | 业内常见 | 差异影响 |
|------|-----------|---------|---------|
| **方法** | 简单时间切分（75/25，单次 split） | 更常用 rolling/expanding window 或 walk-forward | 大 |
| **OOS 检验内容** | 仅检查 Rank IC 和 Q5-Q1 spread 的正负 | 应包含完整的指标对比（IC IR、分层收益、Sharpe） | 中 |

单次 split 的结果对切分点敏感性很高。2019-2021 是小盘成长风格，2022-2024 是熊+大盘价值——换一个切分点可能结论完全不同。业内通常用 rolling 60-month 或 expanding window 来降低随机性。

## 优先级判断

### 真正可能改变结论的差异（建议尽快补）

1. **Forward return 没有缩尾** — 极端收益直接污染 IC 和分层
2. **OOS 用单次 split** — 结论对切分点太敏感
3. **组内等权无市值加权版本** — 可能把"小票效应"误判为因子 alpha
4. **Simple technical baseline 没实际运行** — gate 少了一个关键的锚

### 有一定影响但不紧急

5. 中性化用 OLS 而非 WLS
6. ST 用名称匹配而非专用字段
7. `new_stock_window_days` 默认 20 天偏短
8. 牛熊分层 IC 缺失
9. Q5-only 回测缺少多空组合和纯因子组合

### 基本 OK

10. MAD 缩尾参数和方法
11. 标准化 ddof 选择
12. 收益计算公式
13. IC / 分层 / 换手计算方法
14. 噪声基线实现
15. 行业暴露和市值分段 IC
