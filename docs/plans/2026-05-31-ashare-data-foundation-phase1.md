# A股日频数据底座一期计划

## 任务名

先把 A 股日频量价多因子策略的数据底座搭起来。

## 目标

- 用 Tushare Pro 拉取基础日频数据
- 把原始数据保存下来
- 建出一版本地数据库/数据仓库
- 生成一张 `daily_panel`
- 输出基础数据质量检查报告

这次不做：

- 因子库
- 回测系统
- 策略系统
- dashboard
- 全历史补数

## 当前范围

- 只验证最近 5 个交易日
- 只做日频基础数据底座
- 只用 Tushare Pro 作为数据源

## 实现思路

1. 用 `trade_cal` 找最近 5 个交易日
2. 拉取这些交易日对应的基础表
3. 原始结果先落地保存
4. 再写入本地数据库
5. 基于标准表生成 `daily_panel`
6. 对结果做基础质检并输出报告

## 主要产物

- 原始数据文件
- 本地数据库文件
- 标准化基础表
- `daily_panel`
- 质检报告

计划包含的主要数据：

- 交易日历
- 股票基础信息
- 日频未复权行情
- 复权因子
- 每日指标
- 停复牌数据
- 涨跌停价格
- 申万行业分类和历史行业归属

## 验收标准

- 最近 5 个交易日可以稳定拉取
- 原始数据能落地保存
- 本地数据库能查询
- `daily_panel` 能生成
- `trade_date + ts_code` 不重复
- 能输出一份看得懂的质检报告

## 风险和注意点

- 申万行业归属不是静态字段，要保留历史区间
- 行业匹配要按 `in_date / out_date / is_new` 处理
- 一期先验证链路，不追求一次做完整个数据库
- 如果 Tushare 权限或接口返回有差异，要在报告里暴露出来
