# 文档约定

想快速了解项目结构和 `src` 文件作用，先看 [PROJECT_MAP.md](PROJECT_MAP.md)。

这个目录只记录“大任务”和重要说明。

## 不记录什么

- 很小的改字
- 很小的字段调整
- 临时排查

## 需要记录什么

- 一个阶段性开发任务开始前的计划
- 一天内完成的重要工作
- 明显影响后续方向的决定

## 使用方式

1. 开始一个大任务前，在 `docs/plans/` 下面新建一个计划文件。
2. 计划文件尽量用简单语言，写清楚目标、范围、做法、验收。
3. 每天完成阶段性工作后，在 `docs/worklog.md` 追加一条记录。
4. 小改动不单独记，合并成“今天完成了什么”来写。

## 命名建议

- `YYYY-MM-DD-任务名.md`

## 记录原则

- 先写人话，再写技术词。
- 先写结果，再写细节。
- 保持短句，方便快速扫读。
